PHASE 1 — FIX PRODUCT LOGIC FIRST
==================================================

Before redesigning the UI, fix the logic model properly.

A. REDESIGN STATUS SEMANTICS
Implement a better status model such as:
- Healthy = active accounts exist and performance is strong
- Degraded = active accounts exist but health is poor
- Unused = registered platform with 0 linked accounts
- Inactive = manually disabled by admin
- Dead = accounts exist but all/most are failed/expired/unusable
- Needs Review = inconsistent data or stale/no recent checks

Make sure:
- status and health score never contradict each other
- a platform with 0 accounts is not shown as 100% healthy and dead
- score meaning and status meaning are clearly separated

B. REDESIGN HEALTH SCORE LOGIC
Define a proper score formula using platform-level signals such as:
- active account ratio
- success/failure rate
- recent verification/check outcomes
- account expiry/failure signals
- active session health
- stale-check penalty
- confidence/coverage of recent checks

If a platform has no accounts:
- do NOT give it 100 health by default
- either mark as Unused with a neutral/empty score state
- or show "No data" instead of fake 100%

C. DEFINE DIFFERENCE BETWEEN:
- registry existence
- activity/usage
- operational health
- admin state (enabled/disabled)

Do not mix them into a single misleading indicator.
