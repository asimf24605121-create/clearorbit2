## 3) Correct dashboard behavior

This is the key requirement.

### Rule A: If access is in minutes or hours

When a user has active access in **minutes** or **hours**:

* after login, user should still see the **normal dashboard/home**
* but do **not** show:

  * give detail page
  * edit profile page
  * forced profile completion flow
  * any mandatory onboarding/profile-completion screen

Important:

* user should not be forced to fill details
* user should be able to use the dashboard and assigned platform access normally
* simply hide or disable detail/profile completion pages for short-access users

### Rule B: If access is in days

When a user has active access in **days**:

* after login, user should see the **normal dashboard**
* edit profile page can be available
* profile/details page can be available
* but still **do not force** profile completion

Important:

* even for day-based users, profile completion must be optional
* no forced redirect to detail form
* no blocking dashboard until profile is completed
